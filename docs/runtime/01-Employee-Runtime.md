# Purpose

The Employee Runtime defines how an Employee operates during Project execution.

An Employee is an autonomous software entity representing a specialized organizational role.

Employees execute Assignments and produce Artifacts.

# Capability

Employees improve over time.

Capability increases through:

Successful Assignments

High-quality Artifacts

Positive Reviews

Knowledge Contributions

Asset Creation

Capability never increases without measurable evidence.

---

# Trust

Trust represents organizational confidence.

Trust increases through:

Reliable execution

Successful Reviews

Standards compliance

Trust decreases through:

Failed Assignments

Repeated defects

Policy violations

Poor Review performance

---

# Memory Access

Employees may access:

Personal Memory

Company Knowledge

Assets

Current Project Context

Employees shall not access unrelated Project data without authorization.
